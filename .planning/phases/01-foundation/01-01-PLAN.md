**MainWindow.xaml**: Configure as a transparent overlay window:
```xml
<Window x:Class="TaskbarPet.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"

                   HorizontalAlignment="Center" VerticalAlignment="Center"/>
    </Border>
</Window>
```

Key properties explained:
- `AllowsTransparency="True"` + `WindowStyle="None"` = layered window (WPF requirement)
- `Background="Transparent"` = transparent base
- `ShowInTaskbar="False"` = no taskbar button
- `Topmost="True"` = stays above taskbar (will be dynamically managed later)
- `ResizeMode="NoResize"` = fixed size
- `SizeToContent="WidthAndHeight"` = window sizes to content

**MainWindow.xaml.cs**: Implement click-through behavior:

Also override `OnSourceInitialized` to add `WS_EX_NOACTIVATE` extended window style (Pitfall 11 prevention):
```csharp
protected override void OnSourceInitialized(EventArgs e)
{
    base.OnSourceInitialized(e);
    var helper = new WindowInteropHelper(this);
    int exStyle = GetWindowLong(helper.Handle, GWL_EXSTYLE);
    SetWindowLong(helper.Handle, GWL_EXSTYLE, exStyle | WS_EX_NOACTIVATE);
}
```

Add P/Invoke for `GetWindowLong` and `SetWindowLong`:
```csharp
private const int GWL_EXSTYLE = -20;
private const int WS_EX_NOACTIVATE = 0x08000000;

[DllImport("user32.dll")]
static extern int GetWindowLong(IntPtr hWnd, int nIndex);

[DllImport("user32.dll")]
static extern int SetWindowLong(IntPtr hWnd, int nIndex, int dwNewLong);
```

Position the window initially at the bottom-center of the primary screen's working area (temporary positioning — will be replaced by taskbar-aware positioning in Plan 03):

```csharp
public MainWindow()

    Top = workArea.Bottom - Height - 10;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Window → OS | Transparent window with click-through; untrusted input from mouse events |
| App → Windows API | P/Invoke calls to user32.dll for window style manipulation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | WS_EX_NOACTIVATE style | mitigate | Use `WS_EX_NOACTIVATE` to prevent window from receiving focus, reducing attack surface for focus-based injection |
| T-01-02 | Elevation of Privilege | P/Invoke user32.dll calls | mitigate | Only call `GetWindowLong`/`SetWindowLong` with `GWL_EXSTYLE` on own window handle; never on other process windows |
| T-01-03 | Spoofing | Transparent overlay window | accept | Window is local-only overlay with no network exposure; spoofing risk is minimal as it only renders on user's own desktop |
</threat_model>
